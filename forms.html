Form Components

This page documents the recommended markup structure to build the Balihoo form builder forms and fields. Field types, as defined by the form builder model, will each have their own css component to make maintaining and updating styles easy. Components will be defined using css classes. In some cases components will need subcomponent classes to denote alterations to the base component. An example of this is with the select and multiselect elements which may change from radios to checkboxes or dropdowns based on how the field is configured. subcomponent classes will need to be applied in addition to the component classes.

Component classes are built starting with our universal balihoo namespace bh--, which is follow by form- and then the component name. For example bh--form-text or bh--form-color. Subcomponent classes extend their main component name. For example a subcomponent of the select component is bh--form-select-radios.

Requirements

  • Balihoo Bootstrap
  • Bootstrap JS

    If you are including individual bootstrap js modules, you will need the following...

    • Transition
    • Collapse
    • Dropdown
    • Modal
    • Carousel
  • Spectrum Color Picker (https://github.com/bgrins/spectrum)
  • Time Picker (https://github.com/acpmasquerade/bootstrap3-timepicker2)
